# 订单管理模块用户操作指南

## 概述
订单管理模块为管理员提供完整的订单管理功能，包括订单查询、筛选、详情查看、邮件重发和退款操作。

## 功能特性

### 1. 订单列表查看
- 显示所有订单的基本信息
- 支持分页浏览，每页显示20条记录
- 实时刷新订单状态

### 2. 订单筛选
支持多维度筛选：
- **订单状态**：待支付、已支付、已发货、已完成、已退款、失败、已取消
- **支付方式**：支付宝、Creem
- **日期范围**：支持自定义开始和结束日期
- **关键词搜索**：可搜索订单ID或用户邮箱

### 3. 订单详情查看
点击订单详情按钮可查看：
- 基本信息（订单ID、商品、用户邮箱、状态、金额）
- 支付信息（支付时间、网关订单ID）
- 发货信息（发货时间、退款时间）

### 4. 邮件重发
- 对已发货或已支付订单可执行邮件重发操作
- 系统会重新发送包含订单详情和商品信息的邮件
- 需要管理员确认后执行

### 5. 订单退款
- 对已支付订单可执行退款操作
- 需要输入退款原因
- 系统会调用支付网关执行退款
- **注意：退款操作不可撤销**

## 操作步骤

### 1. 访问订单管理页面
1. 登录管理员后台
2. 在首页点击"订单管理"卡片或通过左侧导航菜单进入
3. 进入订单管理页面

### 2. 筛选订单
1. 使用页面顶部的筛选器组件
2. 选择需要筛选的条件：
   - 订单状态
   - 支付方式
   - 日期范围
3. 点击"应用筛选"按钮
4. 点击"重置"可清除所有筛选条件

### 3. 查看订单详情
1. 在订单列表中点击"详情"按钮
2. 在弹出的详情模态框中查看完整订单信息
3. 点击"✕"或点击模态框外区域关闭

### 4. 重发邮件
1. 在订单列表中找到已发货或已支付订单
2. 点击"重发邮件"按钮
3. 在确认对话框中检查订单信息
4. 点击"确认重发"执行操作
5. 等待系统返回操作结果

### 5. 订单退款
1. 在订单列表中找到已支付订单
2. 点击"退款"按钮
3. 在确认对话框中输入退款原因（必填）
4. 检查订单信息摘要
5. 点击"确认退款"执行操作
6. **注意**：此操作不可撤销，请谨慎操作

## 注意事项

1. **权限限制**：只有已认证的管理员可以执行这些操作
2. **操作限制**：
   - 仅已发货或已支付订单可以重发邮件
   - 仅已支付订单可以退款
3. **退款风险**：退款操作不可撤销，请务必确认订单信息和退款原因
4. **网络问题**：如果操作失败，请检查网络连接或稍后重试

## 常见问题

### Q: 邮件重发失败怎么办？
A: 请检查：
- 订单状态是否为已发货或已支付
- 用户邮箱是否正确
- 网络连接是否正常
- 邮件服务是否可用

### Q: 退款失败怎么办？
A: 请检查：
- 订单状态是否为已支付
- 支付网关是否可用
- 退款原因是否填写
- 网络连接是否正常

### Q: 可以批量操作吗？
A: 目前不支持批量操作。每个订单需要单独操作。

### Q: 如何导出订单数据？
A: 目前不支持导出功能，可在后续版本中添加。
