# 变更：创建下单流程页面

## 为什么
根据项目需求文档，AutoShip自动发货系统需要完整的用户购买流程。在商品展示页面之后，用户需要一个下单页面来收集邮箱、选择商品并创建订单。这是用户端购买流程中的第2步，连接商品展示和支付页面，完成从用户选择到订单创建的完整闭环。

## 变更内容
- 新增下单流程页面（`/checkout`），实现邮箱收集、商品确认和订单创建功能
- 页面接收来自商品详情页的查询参数（`productId`, `productName`, `price`, `currency`）
- 集成后端订单创建API接口
- 优化用户购买流程体验，确保数据在不同页面间的正确传递

## 影响范围
- **影响的功能模块：** 用户端购买流程
- **影响的技术组件：** 前端React页面、后端API路由、订单数据模型
- **依赖关系：** 依赖商品展示页面（已完成），被支付集成页面依赖
- **相关文件：**
  - 前端页面：`frontend/src/pages/CheckoutPage.tsx`
  - 后端API：`backend/src/routes/checkout.ts`
  - 类型定义：`frontend/src/types/order.ts`
- **预计开发工作量：** 约6K tokens

## 预期成果
用户能够在商品详情页点击"立即购买"后，进入下单流程页面：
1. 看到已选商品的确认信息
2. 输入邮箱地址
3. 点击"确认下单"按钮创建订单
4. 成功创建订单后自动跳转到支付页面
