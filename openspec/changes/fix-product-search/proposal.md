# 变更: 修复商品管理页面搜索功能

## 为什么
用户反馈后台管理系统中的商品管理页面无法搜索商品。通过对比分析发现：
- 库存管理页面支持实时搜索（输入后自动触发，带500ms防抖）
- 商品管理页面只有手动搜索（需要点击"搜索"按钮）
- 这导致用户体验不一致，且用户不知道需要点击按钮才能搜索

## 什么变化
**受影响系统**: 前端商品管理页面

**具体改动**:
1. 在 `AdminProductManagement.tsx` 中添加 `useEffect` 监听 `searchTerm` 变化
2. 实现带防抖的自动搜索功能（500ms延迟，与库存管理保持一致）
3. 当搜索词变化时自动重新加载商品列表
4. 保持现有的手动搜索按钮功能不变

**兼容性**:
- 仅前端改动，不影响后端API
- 保持现有手动搜索功能作为备用方案
- 用户体验提升：输入即搜索，无需点击按钮

## 影响
**受影响的规范**: product-management
**受影响的代码**:
- `/frontend/src/pages/AdminProductManagement.tsx` - 添加自动搜索功能

**预期结果**:
- 商品管理页面支持实时搜索
- 与库存管理页面的搜索行为保持一致
- 提升管理员操作效率
