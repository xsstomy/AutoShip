# 变更：添加后台管理系统登录与权限控制

## 为什么
根据项目规划，AutoShip系统需要为管理员提供后台管理功能，包括商品管理、库存管理和订单管理。当前系统虽然已有基础的API访问控制（security规格），但缺少完整的管理员认证和权限管理机制。

为了实现安全的后台访问，必须提供：
1. 管理员登录界面，支持口令验证
2. 会话管理机制，确保登录状态持久化
3. 权限验证中间件，保护管理功能
4. 后台访问控制，防止未授权访问

## 改变什么
- 新增 **admin-auth** 能力，实现管理员登录、权限验证和会话管理
- 前端：添加管理员登录页面，支持用户名/密码登录
- 后端：新增管理员认证API端点，包括登录、登出、权限验证
- 中间件：为管理端API添加权限验证中间件
- 会话管理：使用JWT令牌或安全cookie维护登录状态
- 安全增强：限制登录尝试次数、添加会话超时、记录管理员操作日志

## 影响范围
- **影响规格**：security（增强管理员API认证机制）
- **影响代码**：
  - 前端：`src/pages/admin/Login.tsx`、`src/components/admin/*`
  - 后端：`routes/admin.ts`、`middleware/auth.ts`
  - 数据库：admin_users表设计
- **安全影响**：需要安全存储管理员凭据、实施严格的访问控制
- **依赖关系**：需在实现其他管理模块（商品、库存、订单）之前完成

## 架构考虑
- 使用JWT令牌进行无状态会话管理
- 管理员密码使用bcrypt进行哈希存储
- 实施角色基础访问控制（RBAC）为未来扩展做准备
- 前端路由守卫确保未授权用户无法访问管理页面
