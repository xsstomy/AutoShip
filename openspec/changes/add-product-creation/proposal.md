# 变更: 商品管理页面添加"添加商品"功能

## 为什么
当前商品管理页面只支持查看和编辑现有商品的价格，但缺少创建新商品的核心功能。为了完善商品管理系统，管理员需要能够：

1. **添加新产品**：当业务扩展需要销售新的数字商品时
2. **灵活配置**：设置商品名称、描述、发货类型等基本信息
3. **定价管理**：在创建商品时直接设置 CNY 和 USD 价格
4. **库存初始化**：为新商品设置初始库存或使用模板文本

这将使 AutoShip 系统从单一商品模式扩展到支持多商品的完整电商平台。

## 什么发生了变化
- **后端 API**：在 `admin-products.ts` 中添加 `POST /products` 端点
- **前端界面**：在商品管理页面添加"添加商品"按钮和创建表单
- **数据验证**：添加商品信息的前后端验证逻辑
- **用户体验**：提供直观的商品创建流程和即时反馈

## 影响
- **受影响的规格**：
  - 扩展 `admin-product-management` 规格，添加商品创建能力
  - 与 `database-model` 规格中的商品数据结构保持一致
  - 继续依赖 `admin-auth` 规格的管理员认证功能

- **受影响的代码**：
  - `Backend/src/routes/admin-products.ts` - 添加创建商品 API 端点
  - `frontend/src/pages/AdminProductManagement.tsx` - 添加创建商品界面
  - `Backend/src/services/product-service.ts` - 已有 `createProduct` 方法，需要集成
  - 可能需要更新 TypeScript 类型定义

- **依赖关系**：
  - 依赖现有的 `manage-product-information` 变更（已完成）
  - 与 `add-admin-auth` 变更协同工作（管理员认证）
  - 与库存管理和订单系统集成

## 非目标
- 不实现商品批量导入功能（已有独立的库存导入功能）
- 不实现商品分类和标签系统（后续版本考虑）
- 不实现商品图片上传功能（后续版本考虑）
- 不实现商品版本控制和历史记录（后续版本考虑）
