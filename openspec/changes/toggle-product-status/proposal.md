# 商品上架/下架功能提案

## 变更概述

为商品管理系统添加商品上架和下架功能，允许管理员通过简单的界面操作控制商品的展示状态。

## 问题背景

当前商品管理系统虽然已经有 `is_active` 字段来控制商品状态，但缺少直接的上架/下架操作界面。管理员需要通过编辑商品信息或直接修改数据库来控制商品状态，这不够直观和便捷。

## 建议解决方案

在商品管理界面添加专用的上架/下架按钮和状态显示，让管理员可以：

1. 直观查看商品当前状态（已上架/已下架）
2. 一键切换商品状态
3. 批量操作多个商品的状态
4. 查看状态变更历史

## 涉及范围

- **前端**: 商品管理页面添加状态切换组件
- **后端**: 新增状态更新 API 端点
- **数据库**: 利用现有 `is_active` 字段
- **用户体验**: 提供视觉反馈和确认对话框

## 成功标准

1. 管理员可以在商品列表中看到每个商品的当前状态
2. 点击上架/下架按钮可以切换商品状态
3. 操作有确认提示防止误操作
4. 状态变更后立即更新界面显示
5. 支持批量状态操作
6. 操作成功后显示明确的成功消息

## 风险与约束

- **低风险**: 基于现有的 `is_active` 字段，不需要数据库结构变更
- **权限**: 需要确保只有管理员有权限操作商品状态
- **业务影响**: 下架的商品将不会在前台显示，可能影响销售
